
# 寄存器
## x86-32通用寄存器
- 8个32位寄存器：eax, ebx, ecx, edx, esi, edi, esp, ebp
- 8个16位寄存器：ax, bx, cx, dx, si, di, sp, bp
> 均为对应32位寄存器的低16位
- 8个8位寄存器：ah, al, bh, bl, ch, cl, dh, dl
> 分别为ax-dx的高8位和低9位，h-high，l-low

## x86-64通用寄存器
- 16个64位寄存器：rax, rbx, rcx, rdx, rbp, rsi, rdi, rsp, r8, r9, …, r15
- 16个32位寄存器：eax, ebx, …, esp, r8d, …, r15d
> 分别为对应64位寄存器的低双字(dword)
- 16个16位寄存器：ax, …, sp, r8w, …, r15w
> 分别为对应64位寄存器的最低字(word)
- 20个8位寄存器：al, bl, cl, dl, bpl, sip, dil, spl, r8b, …, r15b, ah, bh, ch, dh
> 前16个寄存器为对应64位寄存器的最低字节(byte)，ah-dh和x86-32相同
> 不能同时使用al-ah, …, dl-dh

## 标志寄存器
### SF(Sign Flag)
运算结果的最高二进制位为1，则SF=1，否则SF=0
### CF(Carry Flag)
运算时最高位向前进位，则CF=1，否则CF=0
### ZF(Zero Flag)
运算结果为0，则ZF=1，否则ZF=0
### OF(Overflow Flag)
加法执行前加数最高二进制位相同，且与结果最高位相反，则OF=1，否则OF=0

# 寻址方式

## 立即寻址（Immediate Addressing）

### 概念

操作数 **直接包含在指令中**，CPU 不需要再访问寄存器或内存。

### 示例

```asm
mov rax, 5
add rbx, 10
```

### 说明

- 操作数是常数
    
- 不需要访问内存
    
- 执行速度快
    


## 寄存器寻址（Register Addressing）

### 概念

操作数 **存放在寄存器中**。

### 示例

```asm
mov rax, rbx
add rax, rcx
```

### 说明

CPU 直接从寄存器读取数据。


### 直接寻址（Direct Addressing）

### 概念

**指令中直接给出操作数所在的内存地址**。

### 示例

```asm
mov rax, [0x600000]
```

### 地址计算

```
EA = 指令中的地址
```

EA = Effective Address（有效地址）

## 寄存器间接寻址（Register Indirect Addressing）

### 概念

**寄存器中存放的是内存地址**。

CPU 通过寄存器中的地址访问内存。

### 示例

```asm
mov rax, [rbx]
```

### 地址计算

```
EA = (RBX)
```

含义：

```
rax = Memory[rbx]
```


## 基址寻址（Base Addressing）

### 概念

使用 **基址寄存器 + 偏移量** 来计算内存地址。

### 地址计算

```
EA = Base + Displacement
```

### 示例

```asm
mov rax, [rbx + 8]
```

### 含义

```
rax = Memory[rbx + 8]
```

### 使用场景

常用于：

- 结构体访问
    
- 栈变量访问
    
- 数据结构
    

## 变址寻址（Indexed Addressing）

### 概念

使用 **变址寄存器 + 位移量** 来计算地址。

### 地址计算

```
EA = Index + Displacement
```

### 示例

```asm
mov rax, [rsi + 16]
```

### 使用场景

主要用于 **数组访问**。

例如：

```c
a[i]
```


## 基址变址寻址（Base + Index Addressing）

### 概念

地址由 **基址寄存器和变址寄存器共同计算**。

### 地址计算

```
EA = Base + Index × Scale + Displacement
```

其中

|参数|含义|
|---|---|
|Base|基址寄存器|
|Index|变址寄存器|
|Scale|比例因子 (1,2,4,8)|
|Displacement|位移|

### 示例

```asm
mov rax, [rbx + rsi*4 + 8]
```

### 说明

```
rax = Memory[rbx + rsi*4 + 8]
```

### 典型用途

数组访问：

```c
int a[10];
a[i]
```

如果 `int = 4字节`

```
地址 = base + i × 4
```


## x86-64 内存寻址统一公式

x86 架构的内存地址几乎都可以表示为：

```
EA = Base + Index × Scale + Displacement
```

说明：

| 元素           | 是否必须    |
| ------------ | ------- |
| Base         | 可选      |
| Index        | 可选      |
| Scale        | 1/2/4/8 |
| Displacement | 可选      |

示例：

```
[rax]
[rax+8]
[rax+rbx]
[rax+rbx*4]
[rax+rbx*4+8]
```


## 总结表

| 寻址方式    | 示例              | 地址计算                  |
| ------- | --------------- | --------------------- |
| 立即寻址    | `mov rax,5`     | 操作数在指令中               |
| 寄存器寻址   | `mov rax,rbx`   | 操作数在寄存器               |
| 直接寻址    | `[0x600000]`    | EA=Address            |
| 寄存器间接寻址 | `[rbx]`         | EA=(RBX)              |
| 基址寻址    | `[rbx+8]`       | Base+Disp             |
| 变址寻址    | `[rsi+16]`      | Index+Disp            |
| 基址变址寻址  | `[rbx+rsi*4+8]` | Base+Index×Scale+Disp |
# 常用机器指令

## 数据传送指令

### 一般传送
`mov opd, ops`
ops为源，opd为目的

### 有符号数传送
`movsx r16/r32, ops/非立即数`
传送时在高位补足符号位

### 无符号数传送
`movzx r16/r32, ops/非立即数`
传送时在高位补足0

### 传送偏移地址
`lea r32, m32`

`lea ecx, [eax+ebx*8+0ch] //将eax+ebx*8+0ch赋给ecx`


### 进栈
`push ops`
栈顶在低地址，进栈时栈寄存器先减再传送
```
//字入栈
esp-2 -> esp
字数据 -> [esp]

//双字入栈
esp-4 -> esp
双字 -> [esp]
```
### 出栈
`pop opd`

```
[esp] -> opd
esp+2/4 -> esp
```

## 算术运算指令

### 加法
```
//加1
inc opd:
opd+1 -> opd

//加
add opd, ops:
opd+ops -> opd

//带进位加
adc opd, ops:
opd+ops+cf -> opd
```

### 减法
```
//减1
dec opd:
opd-1 -> opd

//求补
neg opd:
~opd+1 -> opd

//减
sub opd, ops:
opd-ops -> opd

//带借位减
sbb opd, ops:
opd-ops-cf -> opd
```

### 乘法
opd与ops类型相同
```
//有符号
imul opd, ops:
opd*ops -> opd

imul opd, ops, n:
ops*n -> opd

imul ops:
字节: al*ops -> ax
字: ax*ops -> dx,ax
双字: eax*ops -> edx, eax

//无符号
mul ops:
字节: al*ops -> ax
字: ax*ops -> dx,ax
双字: eax*ops -> edx, eax
```

### 除法与符号扩展
```
//有符号
idiv ops
//无符号
div ops

字节: ax/ops -> al(商),ah(余)
字: (dx,ax)/ops -> ax(商),dx(余)
双字: (edx,eax)/ops -> eax(商),edx(余)

//字节转字
cbw: al中符号扩展至ah

//字转双字
cwd: ax符号扩展至dx

cwde: ax有符号数扩展为32位传送至eax

//双字转四字
cdq: eax扩展为64位送至(edx,eax)
```

### 按位运算
```
//求反
not opd:
~opd -> opd

//按位与
and opd, ops:
opd&ops -> opd

//按位或
or opd, ops:
opd|ops -> opd

//按位异或
xor opd, ops:
opd^ops -> opd

//测试指令（只影响标志位）
test opd, ops:
opd&ops
```

### 移位
```
语句格式： 操作符 opd, n/cl
只允许使用立即数或cl寄存器

//算数左移 shift arithmetic left
sal opd, n

//逻辑左移 shift logical left
shl opd, n

//算数右移
sar opd, n

//逻辑右移
shr opd, n
```

# 转移指令与分支语句
*(注：以下标志位解释：ZF=零标志，CF=进位标志，SF=符号标志，OF=溢出标志，PF=奇偶标志)*

## 基础比较与辅助指令
这些指令用于设置标志寄存器（EFLAGS），为后续的条件指令提供依据。

*   **CMP (Compare - 比较指令)**
    *   **语法:** `CMP opnd1, opnd2` (如 `CMP EAX, EBX`)
    *   **功能:** 执行隐含的减法操作 `opnd1 - opnd2`，不保存结果，但会根据结果修改标志位（CF, ZF, SF, OF, PF）。
*   **TEST (逻辑比较指令 - 课件Slide 27出现)**
    *   **语法:** `TEST opnd1, opnd2` (如 `TEST ECX, ECX`)
    *   **功能:** 执行按位与（AND）操作，不保存结果，仅修改标志位（常用于判断寄存器是否为0，`TEST reg, reg` 后若结果为0则 ZF=1）。

## 简单条件转移指令 (基于单个标志位)
这类指令仅关注标志寄存器中的某**一个**特定位。如果条件满足，则执行跳转：**目标地址 EIP = 当前 EIP + 相对位移量**。

| 汇编指令 | 含义 | 触发条件 (标志位) | 语法 |
| :--- | :--- | :--- | :--- |
| **JZ / JE** | 结果为零 / 相等则跳转 (Jump if Zero / Equal) | ZF = 1 | `JZ 标号` |
| **JNZ / JNE**| 结果非零 / 不等则跳转 (Jump if Not Zero / Not Equal) | ZF = 0 | `JNZ 标号` |
| **JS** | 结果为负则跳转 (Jump if Sign) | SF = 1 | `JS 标号` |
| **JNS** | 结果为正则跳转 (Jump if Not Sign) | SF = 0 | `JNS 标号` |
| **JO** | 发生溢出则跳转 (Jump if Overflow) | OF = 1 | `JO 标号` |
| **JNO** | 未发生溢出则跳转 (Jump if Not Overflow)| OF = 0 | `JNO 标号` |
| **JC** | 发生进位则跳转 (Jump if Carry) | CF = 1 | `JC 标号` |
| **JNC** | 未发生进位则跳转 (Jump if Not Carry)| CF = 0 | `JNC 标号` |
| **JP / JPE** | 偶数个1则跳转 (Jump if Parity / Parity Even) | PF = 1 | `JP 标号` |
| **JNP / JPO**| 奇数个1则跳转 (Jump if Not Parity / Parity Odd) | PF = 0 | `JNP 标号` |

## 无符号数条件转移指令 (基于组合标志位)
这类指令专用于**无符号整数**（如 C语言的 `unsigned int`）比较后的跳转。
*(专家注：课件Slide 11中对JAE和JB的文字描述由于多加了连词略有不严谨，以下整理在提供课件原意的基础上，标注了严格的架构定义)*

| 汇编指令 | 含义 | 触发条件 (课件/实际标准等价) |
| :--- | :--- | :--- |
| **JA / JNBE** | 高于 / 不低于且不等于则跳转 (Jump if Above) | `CF=0 且 ZF=0` |
| **JAE / JNB** | 高于或等于 / 不低于则跳转 (Jump if Above or Equal) | `CF=0` *(课件注为 CF=0 或 ZF=1，实际严格为CF=0)* |
| **JB / JNAE** | 低于 / 不高于或等于则跳转 (Jump if Below) | `CF=1` *(课件注为 CF=1 且 ZF=0，实际严格为CF=1)*|
| **JBE / JNA** | 低于或等于 / 不高于则跳转 (Jump if Below or Equal)| `CF=1 或 ZF=1` |

## 有符号数条件转移指令 (基于组合标志位)
这类指令专用于**有符号整数**（如 C语言的 `int`）比较后的跳转。由于有符号数存在正负和溢出，因此判断条件是 `SF` 和 `OF` 的关系。

| 汇编指令 | 含义 | 触发条件 (课件/实际标准等价) |
| :--- | :--- | :--- |
| **JG / JNLE** | 大于 / 不小于且不等于则跳转 (Jump if Greater) | `SF=OF 且 ZF=0` |
| **JGE / JNL** | 大于或等于 / 不小于则跳转 (Jump if Greater or Equal)| `SF=OF` *(课件注带有或ZF=1，实际严格为SF=OF)* |
| **JL / JNGE** | 小于 / 不大于等于则跳转 (Jump if Less) | `SF≠OF` *(课件注带有且ZF=0，实际严格为SF≠OF)*|
| **JLE / JNG** | 小于或等于 / 不大于则跳转 (Jump if Less or Equal) | `SF≠OF 或 ZF=1` |

## 无条件转移指令
不论标志位如何，强制跳转。

*   **JMP (直接转移)**
    *   **语法:** `JMP 标号`
    *   **功能:** `目标 EIP = 当前 (EIP) + 位移量`。属于相对跳转，优点是可以生成位置无关代码（PIC），节省指令字节数。
*   **JMP (间接转移)**
    *   **语法:** `JMP OPD` (OPD 可以是寄存器 `JMP EBX`，也可以是内存 `JMP DWORD PTR [EBX]`)
    *   **功能:** `目标 EIP = 取出的内容(OPD)`。也就是直接替换 EIP 的值。这是实现 `switch-case` 跳转表、C语言函数指针以及多态（虚函数表）的基石指令。

## 带条件的数据传送指令 (CMOVcc 系列)
这是实现无分支优化（Branchless）的核心指令。它不改变控制流（即不管条件成不成立，EIP都顺序指向下一条指令），而是决定**是否执行数据赋值**。

*   **语法:** `CMOVcc r32, r32/m32` (例如: `CMOVGE EAX, ESI`)
*   **限制:** 目标操作数必须是 32 位/64位寄存器，源操作数可以是寄存器或内存（双字）。不支持8位或16位。
*   **功能:** 如果条件 `cc` 成立，则执行 `(r32/m32) -> r32`；如果条件不成立，则什么也不做（相当于NOP）。
*   **对应指令后缀 (`cc`):** 与跳转指令完全一致。
    *   单标志位: `CMOVE/CMOVZ`, `CMOVC`, `CMOVS`, `CMOVO`, `CMOVP` 等。
    *   无符号数组合: `CMOVA`, `CMOVB`, `CMOVAE`, `CMOVBE` 等。
    *   有符号数组合: `CMOVG`, `CMOVL`, `CMOVGE` (如课件里的), `CMOVLE` 等。

## 条件设置字节指令 (SETcc 系列)
通常用于 C 语言中将布尔比较的结果（`true/false`）实体化为一个数值（`1 或 0`）。

*   **语法:** `SETcc OPD` (例如: `SETG CL`)
*   **限制:** 操作数 `OPD` **必须是一个 8 位（字节）寄存器**（如 `AL`, `CL`, `BL`）或 8 位内存单元。
*   **功能:** 如果条件 `cc` 成立，则将目标操作数置为 `1` (`(opd) <-- 1`)；如果条件不成立，则清零 (`(opd) <-- 0`)。
*   **对应指令后缀 (`cc`):** 同样与跳转指令一致。
    *   单标志位: `SETE/SETZ`, `SETNE/SETNZ`, `SETC`, `SETS`, `SETO`, `SETP` 等。
    *   多标志位: `SETA`, `SETB`, `SETG`, `SETL` 等等。
