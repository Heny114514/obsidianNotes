# 2、 写出求长度为n（二进制位数，如8、16、32）的整数（带正负号）的补码表示的计算方法。
1. 不看数据正负号，将数据转换为二进制，补足n位。超出n位则截断，保留最低n位
2. 若数据无符号，即缺省为+号，该数的补码就是1的结果
3. 若数据有负号，则对1的结果逐位求反再加一，保留低n位

# 4、 简述在进行加法运算时，标志位 ZF、SF、CF、OF的设置规则。
ZF：运算结果为0，则ZF=1，否则为0
SF：运算结果最高二进制位为1，则SF=1，否则为0
CF：运算时最高位向前产生了进位，则CF=1，否则为0
OF：加法执行前两个加数的最高二进制位相同，且结果的最高位与其相反，则OF=1

# 5、 已知8位二进制数x1和x2的值，计算[x1]补＋[x2]补（结果还是8位，即一个字节），并指出每次加法运算后标志位SF、ZF、CF、OF的值。【建议采用16进制计算，不容易出错。SF是看结果的最高二进制位】

(1) x1＝＋0110011B；  x2＝＋1011010B   （x1=0x33, x2=0x5A）
```
x1+x2=0x8d
SF=1,ZF=0,CF=0,OF=1
```
(2) x1＝－0101001B；  x2＝－1011101B   （x1=-0x29, x2=-0x5D）
```
x1+x2=0xd6+0xa2=0x78
SF=0,ZF=0,CF=1,OF=1
```
(3) x1＝＋1100101B；  x2＝－1011101B   （x1=0x65, x2=-0x5D）
```
x1+x2=0x65+0xa2=0x07
SF=0,ZF=0,CF=1,OF=0
```
# 6、对于两个无符号数相加，用什么标志位判断产生了溢出？对于两个有符号数相加，用什么标志位判断产生了溢出？【注意，add 指令是不区分有符号加、还是无符号加的；但程序员可以编写语句，用不同的标志位来判断】
无符号数相加用CF判断溢出，有符号数相加用OF判断溢出
# 7、设有如下 C 语句段，运行后显示的结果是什么？并解释为什么？

```
short  x = 0xFFFF;
short  y =-1; 
unsigned short  z =0xFFFF;
printf(“%d %d %d\n”, x, y, z）；
```
结果为
```
-1 -1 65535
```
short为有符号整数，0xffff即为-1的补码表示（$2^{16}+(-1)$），而unsigned short为无符号，0xffff就是65535
